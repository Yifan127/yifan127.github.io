AWS-Elastic Block Storage(EBS)

Posted on 2018-07-02 | In AWS

The EBS storage service is delivered in volumes, which can be attached to an EC2 instance and used just like a disk drive. The EC2 instance must be located within the same availability zones as the EBS volume.
To access the volume, we must format the it and then mount it to the instance filesystem.

list block devices

[ec2-user@ip-172-31-90-214 ~]$ lsblk
NAME MAJ:MIN RM SIZE RO TYPE MOUNTPOINT
xvda 202:0 0 8G 0 disk
└─xvda1 202:1 0 8G 0 part /
xvdb 202:16 0 8G 0 disk

create a filesystem

[ec2-user@ip-172-31-90-214 ~]$ sudo mke2fs /dev/xvdb
mke2fs 1.42.12 (29-Aug-2014)
Creating filesystem with 2097152 4k blocks and 524288 inodes
Filesystem UUID: b6175fcc-1668-416b-a613-ac57909019e0
Superblock backups stored on blocks:
32768, 98304, 163840, 229376, 294912, 819200, 884736, 1605632

Allocating group tables: done
Writing inode tables: done
Writing superblocks and filesystem accounting information: done

mount the volume to a filesystem

[ec2-user@ip-172-31-90-214 ~]$ sudo mount /dev/xvdb /mnt
sudo mount /dev/xvdb /mnt

access the volume

[root@ip-172-31-90-214 ec2-user]# cd /mnt

umount the filesystem

[root@ip-172-31-90-214 /]# umount /mnt
